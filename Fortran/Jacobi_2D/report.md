# 实验报告：采用二维剖分实现Jacobi迭代

## 源代码

使用**Fortran90**语言，我们实现了**并行二维剖分Jacobi迭代**的算法。源代码参见`./main.F90`。

采用namelist文件的方式指定输入参数，其中`N`为进程数，`S`为矩阵规模。我们给出了一个示例文件：`./config.nml`。

程序会将计算结果和运行时长（单位为s）以文件格式保存至工作目录。

## 自动化测试脚本

我们提供了一个能够自动测试**结果正确性**并计算**计算效率**的脚本：`./test.py`。该脚本使用Python语言编写，需要在Python3环境下运行。

## 结果分析

可以看出，在串行计算时（即进程数等于1时），时间消耗随着矩阵规模`S`的增大而增大。当矩阵规模增大至两倍时，时间消耗大约增加至四倍，与理论时间复杂度$O(S^2)$相符。

而在某一固定矩阵规模下，增加进程数`N`，时间消耗降低，加速比相应地有所提高。但随着`N`的增大，进程通信的时间也会增加，导致程序整体的效率有所降低，在`N=8`时的效率甚至不及串行时的$50\%$。这一现象在矩阵规模`S`较小时更为明显，是因为在小规模下通信时间占总时间比例更高（通信时间关于`S`线性提高）。

因此，我们应根据具体矩阵规模的大小，在加速比和效率间取得平衡，从而选择最合适的进程数`N`。
